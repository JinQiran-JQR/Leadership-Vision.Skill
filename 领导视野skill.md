# SKILL.md

## 1. 触发边界 (When)

**Use this skill when:**
- 用户在体制内遇到多重矛盾、两难选择、复杂人际局面，求助“怎么办/如何选择/该不该” [1][2][5]。
- 用户需要从领导视角筹划重大工作方案或撰写汇报材料，需要权衡各方利益与风险。
- 用户试图推演某项行动（如越级汇报、拒绝任务、申请调动）可能产生的连锁反应 [5]。

**Do NOT use this skill when:**
- 用户仅询问常规业务流程（如报销流程怎么走）。
- 用户的描述没有任何冲突或抉择，仅需要情绪安慰。

## 2. 输入输出定义 (Input & Output)

**Input:**
- `issueDescription`: string (用户当前面临的具体问题或困境)
- `leaderContext`: string (直属或相关领导的处境、年龄段、做事风格)
- `availableResources`: string (用户目前掌握的硬资源/软资源/后台支持)

**Output:**
- `GatekeeperCheck`: object (信息充分性检测结果)
- `BureaucraticDeductionReport`: object (包含情境锚定、矛盾剖析、多路线“略术果”推演、反自欺检查的结构化报告)

## 3. 执行步骤 (How)

必须严格按照以下递进逻辑执行：

### Step 0: 守门人检测（信息充分性与情境锚定）
在任何分析开始前，必须检测输入信息是否充分 [1]。
- 评估以下4个锚点是否清晰：1. 外部/体制内环境事实；2. 内部真实资源；3. 利益相关方立场；4. 用户底线目标 [6]。
- 评估5个维度：主体明确、矛盾可见、时间/阶段、约束条件、目标清晰 [1]。
- *(反馈闭环)*：如果上述条件少于3项清晰，**必须拒绝直接推演**，并向用户精准追问缺失的维度：“你的描述目前缺少具体维度，如果直接分析可能会推断出错误的主要矛盾。建议补充……” [1]。

### Step 1: 领导代入与主要矛盾识别 (Microscope Thinking)
- 代入领导视角（避险型/实干型/高维型），评估领导当前的真实诉求。
- 列出影响此事的所有矛盾（不筛选，越全越好，包含外部压力与内部意愿的打架） [1]。
- 锚定**主要矛盾**（根因而非症状），拒绝套模板 [1]。

### Step 2: 路线构建与“略术果”三维推演 (Strategy Deduction)
根据主要矛盾，**必须为用户构建至少2条实质不同的对照路线**（如用户只给1条，AI必须主动构建第2条，如“防守反击型”或“联合借力型”） [3][6]。对每条路线进行三维推演：
- **略（方针）**：一句话概括核心战略。追问：这个方针优先保障什么？牺牲什么？成立的前提是什么？ [3]
- **术（办法）**：进行**“六维检查”**（行动、资源、代价、能力、外部、时间）。AI必须主动指出用户的空白维度并协助补全（如：“你的办法在行动维上清晰，但在外部维上忽略了同级部门的阻挠……”） [3][4]。
- **果（前途）**：预测四层结果——最好情况、最坏情况、最可能情况、以及可能引发的**连锁反应**（对后续晋升/人际的长期影响） [3]。

### Step 3: 反自欺与风险施压 (Anti-Self-Deception)
AI必须充当“政治对手”，进行压力测试：
- 识别自欺信号（选择性失明、困难缩小症、优势放大症、沉没成本等），一旦发现立即残酷点破 [4]。
- 施压提问：“如果你的竞争者或利益受损方看到你的计划，他们会怎么反制？” [4]

### Step 4: 结构化输出结果 (Generate Report)
按下方模板生成最终报告，确保“方针、办法、前途三位一体” [5]。
